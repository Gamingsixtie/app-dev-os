**Status:** prepared, awaiting user trigger
**Date prepared:** 2026-05-05

## Goal

Switch the production deploy of `toolvo.vercel.app` from the upstream `Gamingsixtie/concurrentie-rekentool-VO` repo to `Gamingsixtie/app-dev-os` (this monorepo) with `apps/concurrentoolVO/` as root directory. After flip: `git push origin main` on app-dev-os → Vercel deploys automatically.

## Pre-flight checklist (verify before starting)

- [ ] Local `npm run build` in `apps/concurrentoolVO/` succeeds and output matches what upstream produces (size + behavior smoke-test)
- [ ] All wanted fixes are merged into `app-dev-os/main` via PR from `dev`
- [ ] All env vars currently set in the Vercel project (`toolvo`) are documented somewhere safe — they need to be re-entered in the reconfigured project (Vercel does not migrate env vars when you change the connected repo)
- [ ] User has access to the Vercel dashboard and can do the Git-disconnect/connect themselves
- [ ] Backup plan: if the new deploy breaks something, "Promote previous" in Vercel deployments rolls back to the last working build in ~30s (per runbook.md rollback path 1)

## Steps

gh pr merge --merge

### Phase 2 — Vercel reconfiguration (manual, dashboard)

User does in browser:

1. Open `https://vercel.com/dashboard` → select `toolvo` project
2. **Settings → Git**:
   - Disconnect repository: `Gamingsixtie/concurrentie-rekentool-VO`
   - Connect repository: `Gamingsixtie/app-dev-os`
   - Production Branch: `main`
3. **Settings → Build & Development**:
   - Framework Preset: `Vite`
   - Root Directory: `apps/concurrentoolVO`
   - Build Command: `npm run build` (or leave default if Vite preset fills it)
   - Output Directory: `dist` (Vite default)
   - Install Command: `npm install` (this app uses npm, not pnpm)
   - Node.js Version: 22.x (matches monorepo conventions)
4. **Settings → Environment Variables** — copy ALL existing keys from current production:
   - `ANTHROPIC_API_KEY` (server-side, for /api/* functions)
   - `SUPABASE_URL`
   - `SUPABASE_SERVICE_KEY`
   - `VITE_SUPABASE_URL` (client-side)
   - `VITE_SUPABASE_ANON_KEY` (client-side)
   - `VITE_SENTRY_DSN` (if used)
   - `SENTRY_AUTH_TOKEN` / `SENTRY_ORG` / `SENTRY_PROJECT` (if used)
   - **NEVER** add `VITE_DEV_AUTOLOGIN_*` to production — these are dev-only

### Phase 3 — First deploy from monorepo

Trigger by pushing a tiny commit to main (e.g., a no-op README touch) OR by clicking "Redeploy" on latest commit in Vercel deployments tab.

Vercel will:
- Clone `app-dev-os` repo
- `cd apps/concurrentoolVO` (per Root Directory)
- `npm install` → `npm run build` → publish `dist/`

Watch the build log for failures. Common gotchas:
- Missing env var → build succeeds but app fails at runtime
- Lockfile mismatch → `npm install` complains about peer deps
- Vite manualChunks config not picked up → bundle is bigger than expected (compare to baseline)

### Phase 4 — Validate before considering it done

- Open the new preview/production URL Vercel gives → smoke test the same flows as current production (login, school list, wizard, comparison view)
- Verify `import.meta.env` values are populated (no "VITE_SUPABASE_URL is required" errors in console)
- Verify chunk sizes match local build output (~91 kB main, ~190 kB first-paint gzip)

### Phase 5 — Cutover (only if Phase 4 is green)

If Vercel auto-published the new build to `toolvo.vercel.app` already (because it deployed `main`), you're already cut over.

If you ran a preview first: in Vercel, "Promote to Production" the validated preview deployment.

### Phase 6 — Cleanup (optional)

- **Archive (not delete)** the old upstream `concurrentie-rekentool-VO` GitHub repo. Disable webhooks but keep the repo for history. If you delete it, you lose git blame history that pre-dates the subtree import.
- Update `apps/concurrentoolVO/code_context/runbook.md` deploy section: remove the upstream-deploy note, replace with "monorepo `main` → Vercel"
- Update `apps/concurrentoolVO/AGENTS.md` § Vercel deploy migration: mark as completed with date

## Rollback (if Phase 4 fails)

If the new deploy is broken:

**Fast (Vercel-only, ~30s):** In Vercel deployments → find last green deployment from the OLD repo → "Promote to Production". Site reverts; investigate the new monorepo build at leisure.

**Slow (Git-side, ~5 min):** Reconnect Vercel project to `concurrentie-rekentool-VO` (reverse of Phase 2 step 2). Production builds from upstream as before. App-dev-os monorepo work continues separately until flip is retried.
