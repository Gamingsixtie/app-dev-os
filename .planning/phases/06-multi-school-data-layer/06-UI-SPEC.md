## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, gap between icon and label |
| md | 16px | Default element spacing, card internal padding, form field gaps |
| lg | 24px | Section padding, card padding on desktop |
| xl | 32px | Layout gaps between cards, page horizontal padding |
| 2xl | 48px | Major section breaks, page top/bottom padding |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch targets minimum 44px height (MODE-03 tablet requirement). Applied to all interactive elements: buttons, school cards, search input, wizard navigation.

## School Card Design

Layout: Vertical card, white background, 1px `neutral-200` border, 8px border-radius, 24px padding.

```
+--------------------------------------------------+
| [School Name - Heading 20px semibold cito-primary]|
| [Incomplete badge if applicable]                  |
|                                                   |
| Schooltype: HAVO/VWO          Leerlingen: 1.240  |
| Modules: Cito LVS, DIA Toetsen                   |
|                                                   |
| Laatst bewerkt: 19 mrt 2026                       |
+--------------------------------------------------+
```

Specifications:
- Card width: responsive grid -- 1 column on mobile (<640px), 2 columns on tablet (640-1024px), 3 columns on desktop (>1024px)
- Card gap: 24px (lg token)
- Card min-height: 160px
- Card hover state: `shadow-md` transition, border color shifts to `neutral-400`
- Card focus state: 2px `cito-primary` outline (keyboard accessibility)
- Card is fully clickable (entire card is a link to `/scholen/{slug}/wizard/1`)
- Incomplete profile: amber badge top-right reading "Niet voltooid" in 14px regular weight
- School name: truncate with ellipsis at 1 line
- Module list: truncate with ellipsis at 1 line, show count if overflow (e.g., "Cito LVS +3 modules")
- Metadata (schooltype, leerlingen): 14px, `neutral-500`, displayed as key-value pairs in a 2-column mini grid
- "Laatst bewerkt" timestamp: 14px, `neutral-400`, bottom of card

## Migration Wizard (D-05, D-06, D-08)

Single-step modal overlay when v1 localStorage data is detected on first app load.

### Success State (data found)

```
+----------------------------------------------------------+
|                                                          |
|  Welkom bij de nieuwe versie                             |
|                                                          |
|  We hebben uw eerdere gegevens gevonden.                 |
|  Geef dit schoolprofiel een naam zodat u het             |
|  later gemakkelijk terugvindt.                           |
|                                                          |
|  Schoolnaam                                              |
|  [HAVO/VWO-school_________________________]              |
|  Suggestie op basis van uw eerdere invoer                |
|                                                          |
|           [Annuleren]     [Profiel opslaan]              |
|                                                          |
+----------------------------------------------------------+
```

Specifications:
- Overlay: semi-transparent backdrop `rgba(0,0,0,0.5)`, centered modal
- Modal: white background, 8px border-radius, 32px padding, max-width 480px
- Title "Welkom bij de nieuwe versie": Heading size (20px semibold), `neutral-900`
- Body text: Body size (16px regular), `neutral-700`, 8px below title
- Input field: full width, 44px height, 16px padding, `neutral-200` border, pre-filled with suggested name (D-06)
- Helper text below input: Label size (14px), `neutral-500`, "Suggestie op basis van uw eerdere invoer"
- "Annuleren" button: ghost style, `neutral-700` text, 44px height -- skips migration, starts fresh
- "Profiel opslaan" button: `bg-cito-accent text-white`, 44px height, 16px horizontal padding
- Button row: right-aligned, 24px above bottom padding, 16px gap between buttons
- Input spacing: 24px below body text, 32px below input to button row

### Error State (corrupt data, D-08)

```
+----------------------------------------------------------+
|                                                          |
|  Welkom bij de nieuwe versie                             |
|                                                          |
|  Uw eerdere gegevens konden niet worden overgezet.       |
|  U kunt opnieuw beginnen met een nieuw schoolprofiel.    |
|                                                          |
|                              [Opnieuw beginnen]          |
|                                                          |
+----------------------------------------------------------+
```

Specifications:
- Same modal styling as success state
- No input field
- "Opnieuw beginnen" button: `bg-cito-accent text-white`, single CTA, right-aligned
- Body text conveys the issue without technical jargon (no mention of localStorage, IndexedDB, or migration)

## Incomplete Profile Indicator (D-12)

Badge displayed on school cards when the wizard was not completed through all steps.

- Badge text: "Niet voltooid"
- Badge style: `bg-amber-100 text-amber-700 border border-amber-300`, 14px regular weight, 4px vertical padding, 8px horizontal padding, 4px border-radius
- Position: top-right corner of school card, 8px from top and right edges
- The badge is purely informational -- clicking the card still opens the wizard where the user left off

## Loading & Skeleton States

### School Card Skeleton
- Same card dimensions as `SchoolCard`
- Animated pulse (`animate-pulse` Tailwind class)
- 3 skeleton lines: heading width 60%, body width 80%, metadata width 40%
- Skeleton color: `neutral-200` with `neutral-100` background
- Show 3 skeleton cards during initial Dexie load (typical load <100ms, but covers slow devices)

### School-level Loading (navigating to a school)
- Skeleton version of the wizard shell: progress bar skeleton + content area skeleton
- Shown while Dexie hydrates the Zustand store for the selected school

## Accessibility Notes

- School cards are rendered as `<a>` or `<Link>` elements (not `<div>` with onClick) for keyboard and screen reader support
- Delete dialog uses `role="alertdialog"` with `aria-describedby` pointing to the confirmation body text
- Migration wizard modal uses `role="dialog"` with focus trap
- Search input has `aria-label="Zoek op schoolnaam"`
- Incomplete badge uses `aria-label="Dit profiel is niet voltooid"`
- All skeleton loaders use `aria-hidden="true"`
- Color contrast: all text/background combinations meet WCAG AA (4.5:1 minimum)
