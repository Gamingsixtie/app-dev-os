## Component Inventory

### 1. PriceComparisonChart (Recharts 3 BarChart)

- **Position:** Above the comparison table (first visual element after page title)
- **Type:** Grouped bar chart. Each module = one group. Three bars per group: Cito (#003082), DIA (#9CA3AF), JIJ (#6B7280)
- **Bar sizing:** barSize={32}, barGap={4}, barCategoryGap="20%"
- **Y-axis:** Euro amounts, formatted with euro sign and thousands separator (nl-NL locale)
- **X-axis:** Module short names (from MODULE_CATALOG.name)
- **Tooltip:** On hover, show exact amount formatted as euro and price-per-student. Background white, border 1px #E5E7EB, border-radius 8px, padding 12px. Font 14px/400.
- **Legend:** Below chart. Three items: "Cito", "DIA", "JIJ" with colored squares (8x8px)
- **Missing data:** If a provider has no price for a module, render no bar (gap visible). Tooltip shows "Niet beschikbaar".
- **Height:** Fixed 320px. Width: 100% of container.
- **Responsive:** Below 640px, bars reduce to barSize={20}. Below 480px, module names rotate 45 degrees.

### 2. ComparisonTable

- **Layout:** Rows = modules (grouped by category with category subheader rows). Columns = Module name | Cito | DIA | JIJ
- **Column widths:** Module name 30%, each provider 23.3%
- **Header row:** Background #003082, text white, font 14px/600
- **Category subheader rows:** Background #F8F9FA, text #374151, font 14px/600, left-aligned, spanning full width
- **Data cells:** Each cell shows two lines: line 1 = total cost (16px/600), line 2 = "per leerling" amount (14px/400, text-neutral-500). PriceBadge inline after the total cost.
- **"Niet beschikbaar" cells:** Orange badge (#FF6600 bg at 10% opacity, #FF6600 text), text "Niet beschikbaar", centered in cell
- **Totaalrij:** Bottom row. Background #F9FAFB, font 16px/600 for amounts. Label "Totaal" in first column. No green/red coloring.
- **Verschil indicator:** Below totaalrij, a neutral row showing "Verschil: {amount}" per provider pair. Text #374151, 14px/400.
- **Expand trigger:** Chevron icon (inline SVG, 16x16) in the module name cell. Click expands detail row.
- **Detail row:** Full-width row spanning all columns, background white, border-left 3px solid #003082, padding 24px. Contains: calculation formula, differentiators, editable price fields.
- **Row hover:** Background transitions to #F3F4F6 (200ms ease)

### 3. ModuleDetailPanel (inside expanded detail row)

Three sections stacked vertically with 16px gap:

**Section A: Berekeningsformule**
- Label "Berekeningsformule" (14px/600)
- Formula displayed as: "{aantal leerlingen} x {prijs per leerling} = {totaal}" per provider
- Values use monospace-style rendering (font-variant-numeric: tabular-nums)

**Section B: Onderscheidend vermogen**
- Label "Onderscheidend vermogen" (14px/600)
- Cito advantages first, listed with checkmark prefix: "Cito biedt extra: {list}"
- Competitor advantages below, listed with info icon prefix: "{Aanbieder} biedt extra: {list}"
- If competitor does not offer this module: warning icon + "Deze module wordt niet aangeboden door {aanbieder}"
- Cito items: text #003082. Competitor items: text #374151.

**Section C: Prijs aanpassen**
- Label "Prijs aanpassen" (14px/600)
- Per provider: label (provider name), editable number input (EditableAssumption pattern), current PriceBadge
- Input field: width 120px, height 40px, text-right alignment, euro prefix label outside input
- On value change: PriceBadge switches to [Handmatig], reset button appears
- "Herbereken" button at bottom-right of section. Style: bg-cito-accent, text white, 14px/600, padding 8px 16px, rounded-lg, hover:bg-cito-accent-hover, active:bg-cito-accent-active

### 4. BusinessCaseCTA (below table)

- Full-width banner, background white, border 1px #E5E7EB, border-radius 8px, padding 24px
- Text: "Bekijk de totale waarde inclusief tijdsbesparing" (16px/600, text-cito-accent)
- Arrow icon (inline SVG) after text
- Hover: underline on text, translate-x 4px on arrow (200ms ease)
- Only visible when scenario B data would be available (always render in Phase 2, link to future)

### 5. DisclaimerFooter (reuse from Phase 1)

- Positioned below the BusinessCaseCTA
- No changes from Phase 1 implementation

## Interaction Contract

### Expand/Collapse Detail Rows
- Trigger: Click on chevron icon or entire module name cell
- Animation: height auto-transition, 200ms ease-out. Content fades in (opacity 0 to 1, 150ms)
- Only one detail row open at a time (accordion pattern). Opening one closes the previous.
- Keyboard: Enter/Space on focused row toggles expansion. Focus moves into detail panel on expand.

### Price Override Flow
1. User clicks editable price field in ModuleDetailPanel Section C
2. Input becomes active (EditableAssumption pattern: click-to-edit)
3. User types new value, confirms with Enter or blur
4. PriceBadge updates to [Handmatig] immediately. Reset button appears.
5. "Herbereken" button pulses once (scale 1.0 to 1.05 to 1.0, 300ms) to draw attention
6. User clicks "Herbereken" -- table and chart update simultaneously
7. Loading state during recalculation: 150ms delay guard (no spinner for fast calcs). If > 150ms, show subtle opacity reduction (0.6) on table and chart with "Berekenen..." text overlay

### Reset Override
- Click reset icon next to overridden price
- Value reverts to publication price
- PriceBadge reverts to original status
- "Herbereken" button pulses again

### Chart-Table Sync
- Hovering a bar in the chart highlights the corresponding table row (bg #F3F4F6)
- Clicking a bar in the chart scrolls to and expands the corresponding module detail row
- Smooth scroll (behavior: smooth) with 80px offset for sticky header clearance
