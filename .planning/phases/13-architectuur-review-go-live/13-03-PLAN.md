<task type="auto">
  <name>Task 1: Performance audit -- bundle analysis and build verification</name>
  <files></files>
  <action>
Run a comprehensive performance audit on the production build. This task produces no code changes -- it is a verification task.

1. Build the project and capture output:
```bash
npm run build 2>&1
```

2. Analyze bundle size with vite-bundle-visualizer (per RESEARCH.md recommendation):
```bash
npx vite-bundle-visualizer
```
This generates a `stats.html` file showing a treemap of all bundled dependencies. Open it and check:
- Total bundle size (should be reasonable for an SPA -- ideally under 500KB gzipped for initial load)
- No unexpected large dependencies in the client bundle (xlsx, mammoth, pdf-parse should NOT appear)
- @react-pdf/renderer should be lazy-loaded (check if it appears in a separate chunk)
- recharts is acceptable in client bundle

3. Check individual chunk sizes:
```bash
ls -la dist/assets/*.js | sort -k5 -n -r | head -20
```

4. Verify no serverless-only libraries leaked into client bundle:
```bash
grep -rl "xlsx\|mammoth\|pdf-parse\|@anthropic-ai/sdk" dist/assets/*.js 2>/dev/null && echo "FAIL: serverless libs in client" || echo "PASS: client bundle clean"
```

5. Check for API key leaks in the built output:
```bash
grep -rl "sk-ant\|ANTHROPIC_API_KEY" dist/assets/*.js 2>/dev/null && echo "FAIL: API keys in bundle" || echo "PASS: no API keys in bundle"
```

Report findings. If bundle size is concerning (>1MB gzipped initial), note specific optimization opportunities but do NOT make changes in this task -- this is audit only.
  </action>

  </verify>
