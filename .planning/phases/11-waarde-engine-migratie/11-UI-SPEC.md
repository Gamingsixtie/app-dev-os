# Phase 11 — UI Design Contract

> Visual and interaction contract for the Waarde tab, hero card, time savings table, migration section, multi-year projection chart, and upsell badges. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (hand-built Tailwind components) |
| Icon library | Inline SVG (existing pattern, see EditableAssumption reset icon) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Table cell padding, compact element spacing |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, card internal spacing |
| xl | 32px | Gap between hero card and section blocks |
| 2xl | 48px | Gap between major sections (Tijdwinst / Migratie / Meerjarenprojectie) |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch targets for inline editable fields use min 44px height (h-11) for tablet usability per MODE-03.

### ValueHeroCard Layout

- Full-width white card with `rounded-lg shadow-sm border border-neutral-200`
- Padding: 24px (`p-6`)
- Top line: section title "Totale jaarlijkse waarde" in `text-lg font-semibold text-cito-primary`
- Center: total value in `text-2xl font-semibold text-cito-accent` using `formatCurrency()`
- Bottom row: three sub-values in a horizontal flex with dividers
  - "Prijsverschil: EUR X.XXX" | "Tijdwinst: EUR X.XXX" | "Migratie: EUR X.XXX"
  - Each in `text-sm text-neutral-500`
- When comparison data unavailable: show "Vul eerst de vergelijking in" in `text-sm text-neutral-400 italic`

### TimeSavingsSection Layout

- White card, same styling as hero card
- Uurtarief as `EditableField` in the card header area, right-aligned
- Table uses `text-sm`, alternating row backgrounds (`even:bg-neutral-50`)
- Editable hours column uses the extracted `EditableField` component with dashed underline trigger
- EUR/year column is computed (hours * hourlyRate), right-aligned, formatted with `formatCurrency()`
- Total row: `font-semibold border-t-2 border-neutral-300`

### MigrationSection Layout

- Warning banner: `bg-amber-50 border border-amber-400 rounded-md p-4 mb-4` with amber triangle icon and text "Migratieprijzen zijn indicatief -- vul werkelijke tarieven in"
- Placeholder prices marked with asterisk (`*`) after the amount
- Verschil column: green text for savings, red text for extra costs

### MultiYearSection Layout

- Recharts `BarChart` height 192px (`h-48`), matching existing `MultiYearChart` pattern
- Bar color: `#003082` (cito-primary), bar size 48px
- Chart uses `CartesianGrid strokeDasharray="3 3" vertical={false}`
- Table below chart: compact, no card wrapper, just `mt-4`
- Break-even display: below table, inline with overstapkosten editable field
  - "Terugverdiend in: **X maanden**" or "Geen terugverdientijd" or "Direct terugverdiend"

### UpsellCard Layout (Dashboard)

- White card matching existing dashboard card pattern
- Title: "Upsell-kansen" with count badge
- List of modules with: module name, current provider, savings per student, signal strength dot (green/yellow)
- Each module row links to Vergelijking-tab (`text-cito-primary underline`)

### UpsellBadge Layout (SchoolCard)

- Pill badge matching `SalesSignalBadge` pattern: `inline-flex items-center rounded-full border px-2 py-0.5 text-xs font-semibold`
- Green variant: `bg-green-100 text-green-800 border-green-300`
- Label: "X kansen" (e.g., "3 kansen")
- Position: next to existing pipeline badge on school card

## Interaction Contract

### Inline Editing (TimeSavingsSection, MigrationSection, MultiYearSection)

| State | Behavior |
|-------|----------|
| Display | Value shown with dashed underline, cursor pointer |
| Click/Tap | Switches to `<input type="number">` with value selected |
| Blur | Commits valid value, reverts invalid value |
| Enter | Commits value |
| Escape | Reverts to previous value, exits edit mode |
| After commit | All dependent calculations update reactively (hero card total, table totals, chart, break-even) |

### Tab Navigation

| Trigger | Action |
|---------|--------|
| Click "Waarde" tab | Navigate to `/school/:slug/waarde` |
| Direct URL `/school/:slug/waarde` | Render WaardeTab |
| Back button | Return to previous tab via router history |

### Hero Card States

| State | Display |
|-------|---------|
| All data available | Full hero with total + three sub-values |
| No comparison data | "Vul eerst de vergelijking in" placeholder for prijsverschil sub-value; other sub-values still show |
| No modules selected | All sub-values show EUR 0 |
| Loading | Skeleton pulse on value areas (use `animate-pulse bg-neutral-200 rounded`) |

### Upsell Badge States

| State | Display |
|-------|---------|
| 0 opportunities | Badge not rendered (hidden) |
| 1+ green signals | Green badge with count |
| Mixed green + yellow | Green badge with total count |
| Only yellow signals | Yellow badge with count |
| No moduleSetups filled | Badge not rendered; DashboardTab shows hint "Vul huidige situatie in" |

## Responsive Contract

| Breakpoint | Behavior |
|------------|----------|
| Desktop (>= 1024px) | Full layout as specified above |
| Tablet (768-1023px) | Hero card sub-values stack vertically. Tables remain horizontal with horizontal scroll if needed. Chart height stays 192px. |
| Mobile (< 768px) | Not primary target but functional: all sections stack, tables use horizontal scroll wrapper (`overflow-x-auto`) |

Touch targets: All editable fields and buttons have minimum 44px touch height (`min-h-[44px]`) on tablet.

---

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| Editable fields | `role="button"` + `tabIndex={0}` on display state, matching existing EditableAssumption pattern |
| Tab navigation | `aria-selected` on active tab (existing TabNavigation pattern) |
| Warning banner | `role="alert"` on migration placeholder warning |
| Upsell badge | `title` attribute with full description (e.g., "3 upsell-kansen voor deze school") |
| Chart | `aria-label="Meerjarenprojectie staafdiagram"` on chart container |
| Color meaning | Never convey information by color alone; positive/negative differences also use +/- prefix |
