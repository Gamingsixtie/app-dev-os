### 3. AI Advice — Retentie Variant (modified WizardStep3Advice)

**File:** `src/features/price-comparison/wizard/WizardStep3Advice.tsx`
**Layout:** Three advice sections rendered sequentially in the streaming output area.

| Section | Card Type | Label | Description |
|---------|-----------|-------|-------------|
| Prijsvergelijking | `prijs` | "Prijs" | What the school currently pays vs. competitor offer |

- If empty: schoolplan section is omitted from advice — graceful degradation

**Streaming UX:**
- Identical to existing: blinking cursor (`animate-pulse`) during streaming
- Spinner SVG during initial load before first token
- Same `streamingText` display pattern as current WizardStep3Advice

### 4. Empty/Error States

No new empty states beyond existing wizard empty state ("Selecteer eerst modules..."). Scenario C reuses the same guard.

## Interaction Contracts

### Keuze-UI Flow (ScenarioDetector)

1. User reaches wizard step where `alles-oud-cito` is detected
2. ScenarioDetector renders two option cards instead of the current info banner
3. User clicks one card — selected state applied, confirm button appears
4. User clicks "Doorgaan"
   - **Migratie selected:** `onChooseMigration()` callback fires — navigates to migration view
   - **Concurrent selected:** `onChooseCompetitor()` callback fires — sets `scenario` to `'alles-oud-cito-concurrent'`, wizard continues to step 2

### AI Advice Generation (Scenario C)

1. User reaches step 3 in Scenario C wizard flow
2. "Genereer retentie-advies" button visible (same position as existing "Genereer advies")
3. Click triggers streaming AI call with `scenarioType: 'C'` + optional `schoolplanOpportunities`
4. Streaming text appears with blinking cursor (existing pattern)
5. On completion: parsed advice cards render in three sections (prijs, verlies, zachte deal)
6. Editable matching dropdowns appear below (existing pattern from Phase 16)
7. "Pas tabel aan" button confirms and applies to ComparisonTable

### Responsive Behavior

| Breakpoint | Keuze-UI cards | Table columns | Advice cards |
|------------|---------------|---------------|--------------|
| >= 640px (sm) | Side-by-side row | 2 columns in table | Full width stacked |
| < 640px | Stacked vertically | 2 columns (narrower) | Full width stacked |

All interactive elements maintain `min-h-[44px]` for tablet touch targets (established pattern).
