<objective>
Document upload and AI-powered price extraction: serverless endpoint for parsing PDF/Excel/Word/CSV, client-side upload to Supabase Storage, and diff-view preview for extracted prices.

Purpose: Accountmanagers often receive price lists as documents (PDF offers, Excel spreadsheets). This plan enables them to upload these documents and have the AI extract prices that can be selectively imported into the school profile, maintaining the same confirmation workflow as AI intake.
Output: Vercel serverless document parser, Supabase Storage upload client, DocumentDropzone and DocumentExtractionPreview components, ProductsTab integration, documents bucket creation.
</objective>

    0. **Precondition: Create Supabase Storage bucket for documents** (per RESEARCH.md Pitfall 6):
       - Create `supabase/migrations/create_documents_bucket.sql` with:
         ```sql
         -- Create the documents storage bucket if it doesn't exist
         INSERT INTO storage.buckets (id, name, public)
         VALUES ('documents', 'documents', false)
         ON CONFLICT (id) DO NOTHING;

         -- RLS policy: authenticated users can upload to their school's folder
         CREATE POLICY "Authenticated users can upload documents"
           ON storage.objects FOR INSERT
           TO authenticated
           WITH CHECK (bucket_id = 'documents');

         -- RLS policy: authenticated users can read documents they uploaded
         CREATE POLICY "Authenticated users can read documents"
           ON storage.objects FOR SELECT
           TO authenticated
           USING (bucket_id = 'documents');

         -- RLS policy: service role can download documents (for serverless function)
         CREATE POLICY "Service role can download documents"
           ON storage.objects FOR SELECT
           TO service_role
           USING (bucket_id = 'documents');
         ```
       - Note to executor: If the project uses Supabase CLI migrations, run `npx supabase db push` after creating the file. If not using CLI migrations, apply the SQL directly in the Supabase dashboard SQL editor. Either way, the bucket MUST exist before upload functionality works.

    2. Create `api/extract-document.ts`:
       - Import Anthropic, createClient from '@supabase/supabase-js', pdf-parse, xlsx (as XLSX), mammoth
       - Initialize anthropic and supabaseAdmin same pattern as api/ai-intake.ts
       - Export async function POST(request: Request): Promise<Response>
       - Auth verification: same pattern as ai-intake.ts (Bearer token -> supabaseAdmin.auth.getUser)
       - Parse body: { storagePath: string, fileName: string }
       - Validate: storagePath and fileName are non-empty strings
       - Download file from Supabase Storage:
         ```
         const { data: fileData, error: downloadError } = await supabaseAdmin.storage
           .from('documents').download(storagePath);
         if (downloadError) return new Response('Bestand niet gevonden', { status: 404 });
         const buffer = Buffer.from(await fileData.arrayBuffer());
         ```
       - Extract text based on file extension (per D-14):
         ```
         function extractTextFromFile(buffer: Buffer, fileName: string): Promise<string>
         ```
         - '.pdf': pdfParse(buffer) -> result.text
         - '.xlsx' / '.xls': XLSX.read(buffer, { type: 'buffer' }) -> XLSX.utils.sheet_to_csv for each sheet, join with '\n\n'
         - '.docx': mammoth.extractRawText({ buffer }) -> result.value
         - '.csv' / '.txt': buffer.toString('utf-8')
         - default: throw Error(`Niet-ondersteund bestandsformaat: .${ext}`)
       - Send extracted text to Claude Haiku for price extraction:
         ```
         const response = await anthropic.messages.create({
           model: 'claude-haiku-4-5',
           max_tokens: 2048,
           system: DOCUMENT_EXTRACTION_PROMPT,
           messages: [{ role: 'user', content: `Analyseer het volgende document en extraheer alle prijsinformatie:\n\n${text.slice(0, 10000)}` }],
         });
         ```
       - DOCUMENT_EXTRACTION_PROMPT:
         "Je extraheert prijzen uit documenten voor onderwijstoetsen in het voortgezet onderwijs.\n\nHerkende modules: rekenwiskunde, nederlands, engels, taalverzorging, sociaal-emotioneel, cognitieve-capaciteiten\nHerkende aanbieders: cito, dia, jij\n\nExtraheer per gevonden prijs:\n- moduleId: een van de herkende modules (of 'onbekend')\n- provider: een van de herkende aanbieders (of 'onbekend')\n- amount: bedrag per leerling per jaar in euro's\n- source: waar de prijs vandaan komt (documentnaam, paginanummer als zichtbaar)\n- priceType: 'publication' als het een officiële prijslijst is, 'agreed' als het een offerte of afspraak is\n\nRetourneer een JSON-array van objecten. Als geen prijzen gevonden: retourneer een lege array [].\nGeef ALLEEN geldige JSON terug, geen markdown of uitleg."
       - Parse Claude response: JSON.parse the text content
       - Return JSON response with extracted prices array
       - Error handling: catch all, return Dutch error messages
         - On parse failure: "Geen prijzen gevonden in dit document."
         - On file read failure: "Dit document kon niet worden gelezen."
         - On timeout/network: "Extractie mislukt. Probeer het opnieuw."

    3. Create `src/lib/document-parser.ts`:
       - Export interface ExtractedDocumentPrice { moduleId: string; provider: string; amount: number; source: string; priceType: 'publication' | 'agreed' }
       - Export async function uploadAndExtract(schoolId: string, file: File): Promise<ExtractedDocumentPrice[]>
         a. Validate file type: accept .pdf, .xlsx, .xls, .docx, .csv, .txt. Throw "Niet-ondersteund bestandsformaat" for others
         b. Upload to Supabase Storage:
            ```
            const path = `${schoolId}/${Date.now()}-${file.name}`;
            const { error: uploadError } = await supabase.storage.from('documents').upload(path, file, { cacheControl: '3600', upsert: false });
            if (uploadError) throw new Error('Upload mislukt. Controleer uw internetverbinding en probeer opnieuw.');
            ```
         c. Call serverless function:
            ```
            const headers = await getAuthHeaders();
            const response = await fetch('/api/extract-document', { method: 'POST', headers, body: JSON.stringify({ storagePath: path, fileName: file.name }) });
            if (!response.ok) { const text = await response.text(); throw new Error(text || 'Extractie mislukt'); }
            return response.json();
            ```
       - Import supabase from '@/lib/supabase/client' and getAuthHeaders from '@/lib/ai-intake' (or inline)
       - Per D-14 (Supabase Storage upload then serverless processing)
