```typescript
// Load pricing data from Supabase after auth confirms (per D-03, Gap 1 fix)
useEffect(() => {
  if (SKIP_AUTH) {
    // In dev mode, load immediately (no auth gate)
    usePricingDataStore.getState().loadFromSupabase();
    return;
  }
  if (!user || loading) return;
  // User is authenticated — load pricing data from Supabase
  usePricingDataStore.getState().loadFromSupabase();
}, [user, loading]);
```

- In production: pricing data loads after auth confirms (user is truthy, loading is false)

- The call is via getState() (not a hook) to avoid re-renders — consistent with existing store access pattern per Phase 9 decision
- The effect re-runs if auth state changes (login/logout)
- The store's loadFromSupabase handles errors internally (sets isOffline=true on failure, keeps cached data)
