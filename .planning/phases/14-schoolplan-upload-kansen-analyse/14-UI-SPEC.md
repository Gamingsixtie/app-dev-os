## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 |
| Heading | 18px (text-lg) | 600 (semibold) | 1.2 |
| Display | 20px (text-xl) | 600 (semibold) | 1.2 |
| Label/Micro | 12px (text-xs) | 600 (semibold) | 1.5 |

Notes:
- Two weights only: 400 (normal) for body text, 600 (semibold) for all headings, labels, display, and emphasis
- Body at 14px matches existing UpsellCard, DocumentDropzone patterns
- Heading at 18px matches existing card title pattern (`text-lg font-semibold`)
- Display at 20px for tab section headers only (AI summary header)
- Label/Micro at 12px for KansCard section labels (uppercase tracking-wide)
- All text in formeel Nederlands (u-vorm) per MODE-01

## Interaction Contracts

### Upload Flow
1. User drops/selects file on DocumentDropzone
2. Dropzone immediately disappears, replaced by metadata bar + streaming progress
3. Step 1 indicator activates: "Document wordt samengevat..."
4. Step 1 completes, Step 2 activates: "Kansen worden geidentificeerd..."
5. Step 2 completes, streaming progress disappears, results render
6. No confirmation step required (D-08)

### Replace Flow
1. User clicks "Vervang schoolplan" button
2. Confirmation dialog: "Weet u zeker dat u het schoolplan wilt vervangen? Alle huidige kansen en notities worden verwijderd."
3. Confirm: DocumentDropzone reappears, old data cleared
4. Cancel: nothing happens

### Annotation Flow
1. User clicks "Besproken" or "Niet relevant" toggle on a KansCard
2. Toggle activates immediately (optimistic update)
3. Optional: user types note in note input field
4. User clicks "Notitie opslaan" to persist note
5. Mutation via React Query `useMutation` to update `opportunity_annotations` JSONB

### Error Handling
1. Non-schoolplan document: AI returns `isSchoolplan: false`
   - Show warning card with orange border: "Dit document lijkt geen schoolplan te zijn. Upload een schoolplan om kansen te identificeren."
2. Upload failure: show error in dropzone area (red text)
3. AI analysis failure: show error card with retry button

## State Inventory

| State | Trigger | Visual Treatment |
|-------|---------|------------------|
| Empty (no schoolplan) | No `schoolplan_analyses` row for school | DocumentDropzone + empty state text |
