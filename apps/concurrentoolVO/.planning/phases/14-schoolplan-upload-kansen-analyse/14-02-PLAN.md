1. **Module-level init** (reused across warm invocations):
```typescript
import Anthropic from '@anthropic-ai/sdk';
```

2. **extractTextFromFile(buffer, fileName)** — reuse exact logic from extract-document.ts but only support PDF, DOCX, TXT (NOT xlsx/xls/csv per D-16). Throw for unsupported formats.

3. **Two AI prompts as pure functions:**

`buildSummarizePrompt()` — returns system prompt for step 1:
- Instruct Claude to determine if the document is a schoolplan
- If yes: extract 3-7 key themes/goals, write a 2-3 sentence summary in Dutch
- If no: return `{ isSchoolplan: false, summary: '', themes: [] }`
- Output must be valid JSON matching a subset of SchoolplanAnalysisResult

`buildMatchingPrompt(summary: string, themes: string[], schoolContext?: { levels?: string[], selectedModules?: string[] })` — returns system prompt for step 2:
- Receive themes from step 1
- For each theme, match against MODULE_CATALOG (serialize module names and descriptions into prompt dynamically — NEVER hardcode module names per Research Pitfall 4)
- For each match, check MODULE_DIFFERENTIATORS to find competitor vulnerabilities
- Generate conversation tips in Dutch
- Assign relevance score ('hoog'/'midden'/'laag')
- Include a "Mogelijk ook relevant" section for modules not matching themes but potentially relevant
- School context (levels, selectedModules) helps prioritize if available
- Output must be valid JSON matching the opportunities/alsoRelevant arrays from SchoolplanAnalysisResult

4. **SSE streaming response:**

The POST handler streams Server-Sent Events:
```typescript
const encoder = new TextEncoder();
const stream = new ReadableStream({
  async start(controller) {
    // Send step 1 progress
    controller.enqueue(encoder.encode(`data: ${JSON.stringify({ type: 'step', step: 1, label: 'Document wordt samengevat...' })}\n\n`));

    // Run step 1: summarize
    const config = getModelConfig();
    const summaryResponse = await anthropic.messages.create({
      model: config.model,
      max_tokens: config.maxTokensSummary,
      system: buildSummarizePrompt(),
      messages: [{ role: 'user', content: documentText.slice(0, 30000) }],
    });

    // Parse step 1 result
    const summaryResult = parseSummaryResponse(summaryResponse);

    // If not a schoolplan, send result immediately
    if (!summaryResult.isSchoolplan) {
      controller.enqueue(encoder.encode(`data: ${JSON.stringify({ type: 'result', data: { ...summaryResult, opportunities: [], alsoRelevant: [] } })}\n\n`));
      controller.close();
      return;
    }

    // Send step 2 progress
    controller.enqueue(encoder.encode(`data: ${JSON.stringify({ type: 'step', step: 2, label: 'Kansen worden geidentificeerd...' })}\n\n`));

    // Run step 2: matching
    const analysisResponse = await anthropic.messages.create({
      model: config.model,
      max_tokens: config.maxTokensAnalysis,
      system: buildMatchingPrompt(summaryResult.summary, summaryResult.themes, schoolContext),
      messages: [{ role: 'user', content: JSON.stringify({ summary: summaryResult.summary, themes: summaryResult.themes }) }],
    });

    // Parse and validate against Zod schema
    const analysisResult = parseAnalysisResponse(analysisResponse, summaryResult);
    const validated = SchoolplanAnalysisResult.parse(analysisResult);

    // Send final result
    controller.enqueue(encoder.encode(`data: ${JSON.stringify({ type: 'result', data: validated })}\n\n`));
    controller.close();
  },
});

return new Response(stream, {
  headers: {
    'Content-Type': 'text/event-stream',
    'Cache-Control': 'no-cache',
    'Connection': 'keep-alive',
  },
});
```

6. **Error handling:**
- Auth errors: 401
- Missing storagePath/fileName: 400
- File not found in Storage: 404
- Text extraction failure: 422
- AI call failure: Send SSE error event `{ type: 'error', message: 'Analyse mislukt. Probeer het opnieuw.' }` then close stream

7. **Helper functions for parsing AI responses:**
- `parseSummaryResponse(response)` — extract text content, strip markdown fences, JSON.parse
- `parseAnalysisResponse(response, summaryResult)` — extract text content, strip markdown fences, JSON.parse, merge with summary data
- Both return empty/default values on parse failure (never throw to client)

8. **Document text truncation:** Slice to first 30,000 characters for step 1 (per Research Pitfall 1: large documents).
