# Phase 15 Plan 01: DMU Klantreis Registratie

## Objective

Implement engagement status tracking per DMU contact person, with a decision matrix on the school dashboard, progress indicators on school/kanban cards, stagnation warnings, and DMU status filtering in the school overview.

## Context

@.planning/phases/15-dmu-klantreis-registratie/15-CONTEXT.md
@.planning/phases/15-dmu-klantreis-registratie/15-UI-SPEC.md

## Tasks

### Task 1: Data layer - engagement status types, DB migration, operations
type="auto"

Add engagement status types to models/school.ts, extend Contact interface in db/types.ts with engagementStatus and waitingForContactId fields, create Supabase migration for new columns, add setEngagementStatus operation with system event logging, and extend mapContactRow.

**Done when:**
- ENGAGEMENT_STATUSES const and EngagementStatus type exported from models/school.ts
- Contact interface has engagementStatus and waitingForContactId fields
- Migration file adds columns to contacts table
- setEngagementStatus function in operations.ts logs system events
- mapContactRow maps new fields

### Task 2: UI components - EngagementBadge, DropOffReasonDialog, EngagementStatusSelect, WaitingForSelect
type="auto"

Create EngagementBadge (follows PipelineBadge pattern), DropOffReasonDialog (follows LostDealDialog pattern), EngagementStatusSelect (native select for status changes), and WaitingForSelect (select other contacts for "wacht-op-intern").

**Done when:**
- EngagementBadge renders correctly with all 6 status colors
- DropOffReasonDialog has required reason textarea
- EngagementStatusSelect triggers status change and opens dialog for "afgehaakt"
- WaitingForSelect shows other contacts when status is "wacht-op-intern"

### Task 3: DMU Matrix on Dashboard Tab
type="auto"

Add DmuMatrix component to DashboardTab showing all contacts with columns: Naam, DMU-rol, Bevoegdheid, Status (with EngagementStatusSelect), Wacht op. Include empty state, stagnation badges, and mismatch banner.

**Done when:**
- DMU-beslissingsoverzicht section visible on dashboard
- Matrix shows all contacts with correct columns
- Status changes work inline via select
- Stagnation badge shows for contacts >30 days in same status
- Mismatch banner appears when DMU statuses don't match pipeline
- Empty state shown when no contacts

### Task 4: DMU Progress Indicator on SchoolCard and KanbanView
type="auto"

Create DmuProgressIndicator component showing "DMU X/Y" with optional stagnation dot. Add to SchoolCard badge row and PipelineKanbanView cards.

**Done when:**
- DmuProgressIndicator shows on school cards
- Shows on kanban cards
- Stagnation dot appears when any contact is stagnating
- Returns null when no contacts

### Task 5: DMU Status Filter in SchoolOverview
type="auto"

Add DmuStatusFilter component below FilterBar in SchoolOverviewPage. Filter schools by engagement status (AND logic with pipeline filter). Persist filter in state.

**Done when:**
- DMU filter row visible below pipeline filter
- Filter chips show counts
- AND logic works with pipeline filter
- "Alle" chip clears DMU filter

### Task 6: Build verification
type="auto"

Run npm run build and fix any TypeScript errors. Run tests.

**Done when:**
- `npm run build` succeeds
- `npx vitest run` passes

## Verification

- All 6 engagement statuses render with correct colors
- Status changes persist via Supabase
- Stagnation detection works for >30 days
- DMU matrix visible on school dashboard
- DMU indicators visible on school cards and kanban
- DMU filter works in school overview
- Build passes without errors
