| Token | Value | Usage |

| sm | 8px | Compact element spacing (gap-2, p-2) |
| md | 16px | Default element spacing (gap-4, p-4, mb-4) |
| lg | 24px | Section padding (p-6, mb-6) |
| xl | 32px | Layout gaps (p-8, gap-8) |

| 3xl | 64px | Page-level spacing |

- Touch targets: min-h-[44px] on all interactive elements (buttons, selects, filter chips) -- existing pattern from FilterBar and LostDealDialog

| In gesprek | in-gesprek | `bg-blue-50 text-blue-700 border-blue-200` | Blue -- active engagement, matches "contact-gelegd" |

## Interaction Patterns

### Engagement Status Change Flow

1. User clicks EngagementStatusSelect dropdown in DMU matrix Status column
2. User selects new status from dropdown
3. **If "Afgehaakt" selected:** DropOffReasonDialog opens. User must enter reason (required textarea). Confirm saves status + reason. Cancel reverts dropdown.
4. **If "Wacht op intern" selected:** Status changes immediately. WaitingForSelect appears in "Wacht op" column. User can optionally select who they're waiting for.
5. **All other statuses:** Status changes immediately on selection. Optional note is NOT collected inline (captured via timeline/conversation instead).
6. System event logged automatically for every status change.
7. React Query cache invalidated -- contacts re-fetched for all views.

### DMU Filter Interaction

1. User clicks a DMU status chip in the DmuStatusFilter row
2. Filter applies as AND with existing pipeline filter
3. Schools are shown only if they have at least one contact with the selected engagement status
4. "Alle" chip clears the DMU filter (shows all schools regardless of DMU status)
5. Counts update reactively: each chip shows the number of schools matching that DMU status

### Mismatch Banner Interaction

1. Banner appears automatically when mismatch rules detect inconsistency
2. "Pipeline bijwerken?" link scrolls to / focuses the pipeline status dropdown in the ProfileHeader
3. Banner is informational only -- no dismiss action, it disappears when the mismatch is resolved

## Accessibility Requirements

| Requirement | Implementation |
|-------------|---------------|
| Touch targets | min-h-[44px] on filter chips, select dropdowns, dialog buttons |
| Focus indicators | focus:outline-2 focus:outline-cito-primary focus:outline-offset-2 (existing pattern) |
| ARIA labels | aria-label on icon-only buttons, role="dialog" on DropOffReasonDialog |
| Color contrast | All badge text/background combinations meet WCAG AA (4.5:1 minimum) |
| Keyboard navigation | Tab through filter chips, Enter/Space to activate, Escape to close dialogs |
| Screen reader | Stagnation badge has sr-only text: "{N} dagen in huidige fase" |
