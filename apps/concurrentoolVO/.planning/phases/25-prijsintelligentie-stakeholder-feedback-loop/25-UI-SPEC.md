## Interaction Patterns

### Review Queue (manager-only `/review` page)

1. **List view** — default sorted by newest first. Each item shows provider pill, module name, price diff, submitter name, date, ProposalBadge.
2. **Expand** — click row to expand inline panel with full toelichting, evidence attachment, audit trail.
3. **Approve** — single click "Goedkeuren" button (cito-primary). No confirmation dialog (low-risk action, reversible via new proposal).
4. **Reject** — click "Afwijzen" opens inline textarea for rejection reason (required). Confirm with "Afwijzen" button (destructive red).
5. **Filter** — pill-based filter bar. Multiple filters combine with AND. Active pills use `cito-primary` background, inactive use `neutral-200` border.

### Price Proposal Flow (all roles)

1. **Trigger** — user clicks "Klopt niet?" link next to any publication price in Products tab or comparison view.
2. **Modal** — PriceProposalModal opens with current price pre-filled (read-only). User enters: new price, source, explanation, optional file upload.
3. **AI validation** — on submit, AI normalizes input (matches module-ID, validates amount range). If deviation > 50%, show PriceDeviationWarning before confirming.
4. **Submit** — creates proposal in review queue. Toast confirmation. Modal closes.

### Market Discount Toggle (comparison view)

1. **Default** — "Publicatieprijzen" active.
2. **Toggle** — switches to "Inclusief marktkorting" when pattern data exists. Comparison table and chart recalculate. KortingsPatroonAlert appears above affected providers.
3. **No data** — toggle disabled with tooltip: "Onvoldoende data voor marktkorting-schatting".

### Admin Config Editor (manager-only)

1. **Navigation** — accessible from settings/admin menu.
2. **Per-provider tabs** — each provider shows its pricing strategy structure.
3. **Edit** — inline editable fields for bundle compositions, tier boundaries, package modules.
4. **Validate** — real-time validation that structure is calculator-compatible. Invalid fields highlighted with red border and inline error text.
5. **Save** — saves to Supabase. Success toast. All calculations using this provider auto-refresh.
