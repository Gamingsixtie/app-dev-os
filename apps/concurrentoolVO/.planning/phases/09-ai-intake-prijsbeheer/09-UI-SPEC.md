## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 600 (semibold) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 24px | 600 (semibold) | 1.2 |

Maximum 4 sizes. No additional text roles.

Sub-roles within the 14px Label size:

| Sub-role | Size | Weight | Line Height | Styling | Usage |
|----------|------|--------|-------------|---------|-------|
| Caption | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold` | Price source labels ("Publicatieprijs", "Schoolspecifieke prijs") |
| Section label | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold uppercase tracking-wide` | Uppercase tracking-wide labels above extraction groups (existing ExtractionPreview pattern) |
| Streaming indicator | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold` | Checkmark items during streaming extraction |

All three sub-roles use the same 14px/600 type spec as Label. Visual distinction comes from CSS styling (`uppercase tracking-wide`), not from a separate font size.

## Interaction Patterns

### AI Intake Flow

1. User clicks "+ Gesprek vastleggen" in ConversationsTab
2. ConversationForm opens with IntakeModeToggle defaulting to "Handmatig"
3. User switches to "AI-intake" mode
4. Textarea expands with AI-specific placeholder text
5. User types free-form notes and clicks "Analyseer notities"
6. StreamingExtraction shows progressive field completion via SSE
7. On completion, DiffView replaces StreamingExtraction
8. DiffView shows extracted data with checkboxes (all checked by default for new items)
9. Existing school data shown as read-only reference rows (grey, no checkbox)
10. Conflicts shown in amber with both old and new values
11. User unchecks items they do not want, clicks "Overnemen en opslaan"
12. Selected items are appended to school profile (never overwrite)
13. Conversation record is saved with the original notes text

### Price Management Flow

1. ProductsTab shows each module card with current active price
2. Each module card has "Prijsbeheer" expandable section
3. Expanded section shows PriceHistoryList with radio buttons for active selection
4. Publication price always shown first as reference (dimmed if not active)
5. User clicks "+ Prijs toevoegen" to open PriceEditModal
6. Modal has fields: amount, price_type (radio: publication/agreed), discount_percentage (conditional on agreed), source, verified_at, note
7. On save, new price appears in history list
8. User selects active price via radio button, activation_reason field appears inline
9. Activation reason is required before the radio selection is confirmed
10. Price deviation warning appears inline if amount differs >50% from publication

### Document Upload Flow

1. User clicks "Document uploaden" in ProductsTab header area
2. DocumentDropzone opens below the button (or inline in a collapsible section)
3. Dropzone accepts drag-and-drop or click-to-select, validates file type (PDF/Excel/Word/CSV)
4. On file selection, upload progress indicator shows "Document wordt verwerkt..."
5. Serverless function processes file, returns extracted prices
6. DocumentExtractionPreview shows prices in DiffView format
7. User checks which prices to import, clicks "Prijzen overnemen"
8. Each accepted price creates a SchoolPriceEntry with source = document filename

### Focal Point

The primary visual anchor for each screen:

| Screen | Focal Point | Rationale |
|--------|-------------|-----------|
| ConversationsTab (AI mode) | "Analyseer notities" CTA button (bg-cito-primary, 44px height) | The single action the user is building toward after entering notes |
| ConversationsTab (diff-view) | DiffView checkbox list with "Overnemen en opslaan" CTA at the bottom | The user's attention must scan checkboxes then land on the confirm action |
| ProductsTab (prijsbeheer) | Active price radio indicator (cito-primary filled radio) per module card | The user needs to immediately see which price is active for each module |

## Layout Rules

### ConversationsTab with AI Intake

- ConversationForm renders inline below the "+ Gesprek vastleggen" button (existing pattern)
- IntakeModeToggle is a segmented control at the top of the form: two options, 44px height, rounded-lg
- Manual mode: existing form fields (date, contact, content, tags)
- AI mode: textarea (10 rows min, resizable) + "Analyseer notities" button
- StreamingExtraction renders below the textarea during analysis
- DiffView replaces the streaming display on completion, same width as form
- DiffView sections are collapsible with chevron toggle, all expanded by default

### ProductsTab with Price Management

- Module cards retain existing layout (module name left, price right)
- Each card gains a collapsible "Prijsbeheer" section below the price display
- Expanded section: PriceHistoryList (vertical stack of price entries) + "+ Prijs toevoegen" button
- PriceEditModal uses existing modal pattern (overlay + centered card, max-width 480px)
- DocumentDropzone renders in a dedicated section at the top of ProductsTab, above module cards
- Dropzone has dashed border (border-dashed border-neutral-300), 120px min-height, rounded-lg

### Responsive Behavior

- All layouts are single-column on mobile (< 640px, `max-sm:` prefix)
- Price modal is full-width on mobile with 16px horizontal padding
- DiffView items stack vertically on mobile (checkbox + label above, values below)
- Touch targets remain 44px minimum on all breakpoints
