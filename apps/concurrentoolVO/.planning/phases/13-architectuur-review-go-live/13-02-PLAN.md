The build fails because `supabase.from(mutation.table)` expects a union of valid table names but `mutation.table` is typed as `string`.

Fix by defining a type union of the tables actually used in offline queue operations. From operations.ts, the tables used with `queueIfOffline` are: `schools`, `contacts`, `conversations`, `actions`.

In `src/lib/offline-queue.ts`:

1. Add a type for valid offline-queue table names at the top of the file:
```typescript
type OfflineQueueTable = 'schools' | 'contacts' | 'conversations' | 'actions' | 'school_prices' | 'system_events' | 'schoolplan_analyses';
```

2. Change the `PendingMutation` interface:
```typescript
export interface PendingMutation {
  id: string;
  table: OfflineQueueTable;  // was: string
  operation: 'insert' | 'update' | 'delete';
  payload: Record<string, unknown>;
  timestamp: number;
  conflicted?: boolean;
  conflictReason?: string;
}
```

3. The `supabase.from(mutation.table)` calls on lines ~58, ~83, ~87, ~91 should now type-check because `mutation.table` is a valid table name union. If the Supabase generic still complains, use a type assertion: `supabase.from(mutation.table as any)` -- but try the union type first as it is safer.
