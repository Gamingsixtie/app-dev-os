# Phase 22 — UI Design Contract

> Visual and interaction contract for Architectuur, Testen & Productie-readiness. This phase is infrastructure-focused (testing, CI, error tracking, security, performance). The only new UI surface is the Sentry error boundary fallback screen. All other work is non-visual. Generated by gsd-ui-researcher.

| Property | Value |

## Spacing Scale

Continued from prior phases. No changes. Standard 8-point scale:

| Token | Value | Usage |

## Phase-Specific UI Surface

This phase introduces exactly one new visible UI element: the **Sentry error boundary fallback**. All other work (tests, CI, security, performance) is non-visual.

### Error Boundary Fallback Screen

Displayed when a React component tree crashes in production. Replaces the white screen of death.

| Property | Value |
|----------|-------|
| Layout | Centered vertically and horizontally, max-width 480px |
| Background | `#F8F9FA` (cito-bg) — full viewport |
| Card | White card, rounded-lg, shadow-md, padding 32px |
| Icon | Warning triangle, 48px, color `#FF6600` (cito-accent) |
| Heading | "Er is iets misgegaan" — 20px, semibold, `#111827` (neutral-900) |
| Body text | "De applicatie heeft een onverwachte fout ondervonden. Probeer de pagina te vernieuwen." — 14px, regular, `#6b7280` (neutral-500) |
| Primary button | "Pagina vernieuwen" — calls `window.location.reload()` |
| Secondary link | "Terug naar overzicht" — navigates to `/` |
| Button style | Primary: `#003082` bg, white text, rounded-md, px-16 py-8. Hover: `#1a4a9e` |
| Spacing | Icon to heading: 16px. Heading to body: 8px. Body to button: 24px. Button to link: 16px |

## Non-Visual Contract (Infrastructure)

These elements have no visual component but are documented for completeness:

| Element | Contract |
|---------|----------|
| Sentry init | `enabled: import.meta.env.PROD` only — never captures in dev |
| Sentry DSN | Via `VITE_SENTRY_DSN` env var — app works without it |
| Error boundary wrapper | Wraps `<App />` in `main.tsx` via `Sentry.ErrorBoundary` with `fallback` prop |
| Performance tracing | `tracesSampleRate: 0.1` (10% of transactions) |
| Session replay | `replaysOnErrorSampleRate: 1.0` (all errors), `replaysSessionSampleRate: 0.1` (10% normal) |
| CSP headers | Configured in `vercel.json` — must allow Sentry ingest domain |
