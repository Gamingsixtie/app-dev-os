## New UI Surfaces

### Surface 1: Login Page (`LoginPage.tsx`)

**Layout:** Centered card (max-width 400px) on `bg-cito-bg` full-viewport background. Vertically centered with `3xl` (64px) top offset on desktop, `lg` (24px) on mobile.

**Content structure (top to bottom):**
1. Cito logo or brand mark area (text-cito-primary heading)
2. Page title: "Inloggen" (Heading, 20px, semibold)
3. Subtitle: "Log in om door te gaan naar de rekentool" (Body, 16px, neutral-500)
4. Tab toggle: "E-mail & wachtwoord" | "Magic link" (two options, Label size)
5. Form fields (conditional on selected tab):
   - **Email/password tab:** Email input + Wachtwoord input + "Inloggen" button
   - **Magic link tab:** Email input + "Verstuur magic link" button
6. Error message area (below form, destructive red, Body size)
7. Footer text: "Neem contact op met uw beheerder voor een account" (Label, neutral-500)

**States:**
- Default: form ready, button enabled
- Loading: button shows spinner, inputs disabled
- Error: red error text below form ("Onjuist e-mailadres of wachtwoord" / "Er is iets misgegaan, probeer het opnieuw")
- Magic link sent: success message "Controleer uw inbox — klik op de link om in te loggen" (success green)

**Interaction:**
- Tab toggle switches form content without page reload
- Enter key submits the active form
- Form validation: email format check inline, password required check inline
- After successful login: redirect to school overview (or migration wizard if local data detected)

### Surface 2: Cloud Migration Wizard (`CloudMigrationWizard.tsx`)

**Layout:** Centered card (max-width 500px) on `bg-cito-bg`. Same centering as login page.

**Trigger:** Shown after first successful login when IndexedDB contains school data (D-03).

**Content structure (top to bottom):**
1. Heading: "Uw data overzetten" (Heading, 20px, semibold, cito-primary)
2. Explanation text: "Er zijn {N} school(en) gevonden op dit apparaat. Wilt u deze veilig overzetten naar de cloud? Daarna is uw data overal beschikbaar." (Body, 16px)
3. School list preview: numbered list of school names found in IndexedDB (Label, 14px, max 5 shown, "+N meer" if >5)
4. Progress area (visible during migration):
   - Progress bar: full-width, 8px height, `bg-cito-accent` fill on `bg-neutral-200` track
   - Status text below bar: "School {name} overzetten... ({current}/{total})" (Label, 14px, neutral-500)
5. Action buttons:
   - "Data overzetten" (accent CTA, full width)
   - "Overslaan" (text-only link below, neutral-500, underline)

**States:**
- Ready: shows explanation + school list + action buttons
- Migrating: buttons hidden, progress bar visible, school-by-school status text
- Success: green checkmark icon + "Alle data is succesvol overgezet!" (Body, success green) + "Doorgaan naar overzicht" button (accent CTA)
- Partial failure: amber warning + "{N} scholen overgezet, {M} mislukt" (Body) + expandable error list per failed school + "Opnieuw proberen" button (accent CTA) + "Doorgaan zonder mislukte scholen" text link
- Complete failure: red error + "Er is iets misgegaan bij het overzetten" (Body, destructive) + "Opnieuw proberen" (accent CTA) + "Neem contact op met uw beheerder" (Label, neutral-500)

**Interaction:**
- "Overslaan" requires confirmation: "Weet u het zeker? Lokale data wordt niet overgezet en is alleen op dit apparaat beschikbaar." with "Ja, overslaan" / "Annuleren" buttons
- Progress bar animates smoothly per school
- User cannot navigate away during active migration (no back button)

### Surface 3: Auth Loading State

**Layout:** Full viewport, centered spinner.

**Content:** Cito-primary colored spinner (24px, CSS animation) + "Laden..." text (Label, 14px, neutral-500) below spinner.

**Used when:** AuthProvider is checking session state on app load, or after login redirect.

### Surface 4: Read-Only Indicator (role-based)

**For managers and viewers** who cannot edit:

- Banner at top of school profile: "U bekijkt dit profiel als {role}. Alleen de eigenaar kan wijzigingen aanbrengen." (Label, 14px, `bg-blue-50`, `text-blue-700`, `border-blue-200`, 8px vertical padding, full width)
- All edit buttons, form inputs, and CTA buttons within school profile are visually disabled (`opacity-50`, `cursor-not-allowed`, `pointer-events-none`)
- Delete buttons are hidden entirely (not just disabled)

### Surface 5: Audit Trail Metadata

**"Bijgewerkt door" display** on records that have `updated_by`:

- Format: "{username} - {relative time}" (Label, 14px, neutral-500)
- Shown below: conversation entries, contact cards, school prices, pipeline status changes
- Relative time format: "zojuist", "5 min geleden", "2 uur geleden", "gisteren", "3 mrt 2026" (Dutch)
- If no update: show "Aangemaakt door {username} - {date}" instead

### Surface 6: Owner Badge on School Cards

**In school overview cards/list:**

- Small avatar circle (24px, neutral-200 background, first letter of owner name in neutral-700, Label weight)
- Tooltip on hover: "{owner name} (eigenaar)"
- If current user is owner: no badge (implicit)
- Shown inline with existing school card metadata (next to pipeline badge)

### Surface 7: "Mijn scholen" / "Alle scholen" Filter (D-09)

**In school overview filter bar:**

- Toggle button group: "Mijn scholen" | "Alle scholen" (Label, 14px)
- Default state: "Mijn scholen" active for accountmanagers, "Alle scholen" active for managers/viewers (D-09)
- Active state: `bg-cito-primary`, `text-white`, rounded
- Inactive state: `bg-white`, `text-neutral-700`, `border-neutral-200`, rounded
- Additional filter: "Regio" dropdown (only visible in "Alle scholen" mode) with free-text region values

## Interaction Patterns

### Login Flow
1. User navigates to app URL -> AuthProvider checks session
2. No session -> redirect to `/login` -> LoginPage renders
3. User submits credentials -> loading state -> success: redirect to overview (or migration wizard) / error: show error text
4. Magic link: user enters email -> "Verstuur magic link" -> success message -> user clicks email link -> auto-login -> redirect

### Migration Flow
1. After first login, AuthProvider checks `localStorage('supabase-migration-complete')`
2. If not complete AND IndexedDB has data -> show CloudMigrationWizard instead of app
3. User clicks "Data overzetten" -> progress bar per school -> success/failure state
4. On success -> localStorage flag set -> redirect to school overview
5. On skip -> confirm dialog -> localStorage flag set -> redirect to overview (local data abandoned)

### Role-Based UI
1. AuthProvider fetches user profile (role, team_id) from `users` table after auth
2. Role stored in context, accessible via `useAuth()` hook
3. Components check `userProfile.role` and `school.owner_id === user.id` to enable/disable editing
4. ReadOnlyBanner rendered conditionally in SchoolLayout for non-owners
