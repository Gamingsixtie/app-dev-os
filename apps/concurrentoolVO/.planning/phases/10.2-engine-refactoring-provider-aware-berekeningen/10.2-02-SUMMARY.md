# Phase 10.2 Plan 02: Store Simplification and Legacy Removal Summary

**Simplified store from 6-step pipeline to 3-step pipeline using new calculateComparison() with provider calculators; removed calculateComparisonLegacy; converted parity tests to snapshot regression guards**

## What Was Built

### Task 1: Simplify store to use new engine

Refactored `store.ts` from a 6-step pipeline to a clean 3-step pipeline:

**Removed functions:**
- `applyDiaVolumeDiscount()` -- now inside DIA calculator
- `applyDynamicJijPrices()` -- now inside JIJ calculator
- `applyDiaPackageToResult()` -- now handled by calculateComparison DIA package optimization
- `mergeOverrides()` -- replaced by `Map<string, number>` passed to engine

**Removed imports:** `DEFAULT_PRICES`, `calculateDiaModuleCosts`, `getDiaVolumeDiscountPercent`, `DIA_PACKAGES`, `estimateJijCostPerStudent`, `applyCitoBundlePrices`, `getTotalStudents`

**Simplified `computeExtendedResults()`:** Now takes only `ComparisonResult` (was also taking selectedModules, studentCounts, mergedPrices). DIA package section removed entirely -- `diaPackageResult` comes from `ComparisonResult.diaPackageResult`.

**New pipeline:**
1. `calculateComparison(modules, counts, { citoBundleType, overridePrices })` -- engine handles everything
2. `applyContractPeriodToResult(annualResult, period, citoFactor)` -- post-processing stays
3. `computeExtendedResults(result)` -- hybrid + sensitivity only

Net result: store.ts went from 405 lines to 213 lines (-47%).

### Task 2: Remove legacy function and final validation

- Removed `calculateComparisonLegacy` from `price-comparison.ts`
- Simplified `calculateComparison` signature: removed overloaded `PriceRecord[]` path, now only accepts `ComparisonOptions`
- Converted parity tests to snapshot-based regression tests (30 snapshots for school sizes 100, 500, 800, 1200)
- Updated `price-comparison.test.ts` to use new calculator-based signature throughout
