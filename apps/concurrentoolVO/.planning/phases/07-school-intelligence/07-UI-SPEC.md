# Phase 7 — UI Design Contract

> Visual and interaction contract for School Intelligence: CRM-lite school profiles with contacts, conversations, pipeline, kanban actions, and an enriched school overview. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Tailwind CSS 4 theme, continued from Phase 6) |
| Preset | not applicable |
| Component library | none (hand-built components) |
| Icon library | Inline SVG icons (existing pattern, no icon library installed) |
| Font | System font stack: `ui-sans-serif, system-ui, sans-serif` (existing) |

---

## Spacing Scale

Declared values (continued from Phase 6, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding, tag gaps |
| sm | 8px | Compact element spacing, icon-label gap, timeline connector width |
| md | 16px | Default element spacing, card padding, form field gaps, tab bar gaps |
| lg | 24px | Section padding, card padding on desktop, kanban column padding |
| xl | 32px | Layout gaps between sections, page horizontal padding |
| 2xl | 48px | Page top padding, major section breaks |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch targets minimum 44px height (MODE-03 tablet). Applied to tab buttons, pipeline dropdown, kanban drag handles, action buttons, contact cards, conversation entries.

- Horizontal tabs, bottom-bordered container
- Tab labels: Label size (14px semibold), 44px height, 16px horizontal padding
- Active tab: `text-cito-primary`, 2px bottom border in `cito-primary`

- Tab container bottom border: 1px `neutral-200`

## Accessibility Notes

- Tab navigation uses `role="tablist"` with `role="tab"` and `role="tabpanel"`, `aria-selected`, and keyboard arrow-key navigation
- Pipeline dropdown uses `role="listbox"` with `aria-expanded` and keyboard support
- Contact form slide-over uses focus trap and `role="dialog"` with `aria-label`
- Timeline uses `<ol>` semantic list; system events use `aria-label` describing the event
- Kanban boards use `aria-label` per column ("Te doen, 2 acties"), drag items use `aria-roledescription="sorteerbaar item"` and keyboard instructions
- All modals (LostDealDialog, PipelineReasonDialog, delete confirmation) use `role="alertdialog"` with focus trap
- Pipeline badges use `aria-label` with full status name
- Contact initials circles are decorative (`aria-hidden="true"`), contact name carries the accessible name
- DMU badges use `aria-label` with full position name
- Color alone is never the only indicator: pipeline badges include text labels, timeline uses icons + text
