# Positioning (for in-app messaging)

> Foundation file for **product story inside the app**: feature introductions, value-prop reminders in onboarding, upgrade prompts, why-this-matters copy. Skills load this when writing in-app messaging that explains the product itself.

## What the app is

_One sentence._

Example: _"A team workspace where engineers ship faster by automating the boring parts of code review."_

## What it isn't

_Two or three things people might confuse it with — and the difference._

- **Not** a project management tool — we don't replace Linear/Jira
- **Not** an IDE plugin — we run on your repo, not in your editor
- **Not** an AI code generator — we review what you wrote

## Primary value prop

_The one promise that matters most to the user._

Example: _"Catch bugs before reviewers see them."_

## Three differentiators

_What makes this app distinct? Used in feature-intro copy._

1. _e.g. Works on your existing PRs, no migration_
2. _e.g. Fast — feedback in under 30 seconds_
3. _e.g. Privacy — your code never leaves your infra_

## Hooks for onboarding

_Three sentences that work as in-app intro copy._

- _e.g. "Find issues your tests miss."_
- _e.g. "Catch security holes before they ship."_
- _e.g. "Reviewers see clean PRs."_

## Upgrade prompts (if applicable)

- **What's free**: _scope_
- **What requires upgrade**: _scope_
- **Tone of upgrade prompt**: helpful nudge, never aggressive
